FoodFly

A mobile food-delivery experience — browse restaurants, build a cart, place orders, and open restaurant pages via deep links. Built with Expo, React Navigation, and Zustand.

Features

🍔

Restaurant discovery

Search and filter local restaurants with ratings, tags, offers, and menus.

🛒

Cart & orders

Add dishes from detail screens, adjust quantities, and place orders from the Orders tab.

👤

Auth flow

Register and log in with credentials stored locally (demo mode — no backend).

🚴

Onboarding

Two-step intro that resets navigation into login or home when complete.

🔗

Deep linking

Open a restaurant directly via foodfly://restaurant/:id or Expo dev URLs.

📱

Tab navigation

Home, Search, Orders (with cart badge), and Profile in a bottom tab bar.

Tech stack

Expo ~54 React 19 React Native 0.81 TypeScript React Navigation 7 Zustand AsyncStorage expo-linking

Navigation uses a root stack for onboarding, auth, home, and restaurant details, plus a nested bottom tab navigator inside HomeScreen.

Architecture

Navigation flow

Onboarding1 → Onboarding2 → (reset) LoginScreen | HomeScreen
  LoginScreen ↔ RegisterScreen
  HomeScreen → tabs: Home | Search | Orders | Profile
  Home → RestaurantDetails (stack push or deep link)

Screens are shown or hidden based on isOnboarded and isAuthenticated from the app store. After login, navigation.replace is used so users cannot navigate back to the login screen.

Getting started

Prerequisites

  • Node.js 18+ (LTS recommended)
  • npm or yarn
  • Expo Go on a physical device, or Android Studio / Xcode for emulators

Install & run

# Clone and enter the project
cd FoodFly

# Install dependencies
npm install

# Start the dev server
npm start
# or: npx expo start --clear

Scan the QR code with Expo Go (Android) or the Camera app (iOS). Press a for Android emulator or w for web.

Try the app (demo auth)

  1. Complete onboarding and tap Get Started.
  2. Register a new account, then log in with that email and password.
  3. Browse restaurants, open details, add items, and place an order from the Orders tab.

Deep linking

URL scheme is configured in app.json as foodfly. Restaurant details map to restaurant/:id in App.tsx.

Environment Example URL
Development build / production foodfly://restaurant/resto_001
Expo Go (replace host with yours) exp://192.168.x.x:8081/--/restaurant/resto_001

Open links on the device or emulator where the app runs — not in a desktop browser. On Android emulator: 

adb shell am start -a android.intent.action.VIEW \
  -d "exp://YOUR_IP:8081/--/restaurant/resto_001"

Valid IDs are defined in constatnts/data/restaurants.ts (e.g. resto_001).

State & persistence

Store Key Purpose
app-store app-storage isOnboarded, isAuthenticated, registered userCreds
order-store order-storage Cart items and past order batches

Both stores use Zustand persist with AsyncStorage. Clear app data in Expo Go to reset onboarding and auth state during development.
